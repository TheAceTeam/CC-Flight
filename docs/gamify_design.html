SuperView Gamify Design Run Atlas + Replay Sync / 面向复杂工程 agent log 的游戏化可视化方案

Design Direction

把每次 agent run 变成一张可 replay 的工程地图

SuperView 的游戏化不应该只是积分、徽章或皮肤，而应该让用户用“通关视角”重新理解复杂工程： 用户 prompt 是起点，context 是补给，tool call 是机关，file change 是搭桥，verification 是 checkpoint， 最终输出是通关旗。Run Atlas 负责空间化，Replay Sync 负责把地图和对话、Evidence 同步起来。

1
Run Atlas每轮任务是一座 3D 小岛，一页 timeline 是一片 archipelago。
2
Replay Sync播放地图节点时，同步选中 IM thread 与 Evidence drawer。
3
Context Trail把被传入的文件、工具结果、历史摘要表现成路线上的补给点。

为什么先做这两个

Run Atlas 是用户第一次感到 “wow” 的入口：复杂工程不再是一串 log，而是一段可观察的路线。 Replay Sync 则保证它不是玩具，点击或播放任何节点都能回到真实 log evidence。

视觉语言

保留当前 warm command center 的 UI，3D 场景内部使用开发者能共情的游戏语义： checkpoint、bridge、hazard、finish gate。避免直接依赖受保护 IP，形状和动效表达“平台跳跃感”即可。

MVP 边界

当前先做可切换的 Atlas/Thread 视图、Three.js 场景、节点点击、自动 replay、Evidence 同步。 Context 先以节点和 metadata 表示，后续再扩展成完整 context inspector。

SuperView 数据 Run Atlas 表达 Replay 行为
TaskJourney 一座 task island，大小由事件数、时长、token 用量共同决定。 点击 island 选中该轮任务；若 detail 未加载，先展示摘要路线并后台加载。
user_prompt 起点 block，颜色偏蓝，表示用户输入触发 run。 同步选中用户气泡，Evidence 显示原始 prompt。
tool_call / tool_result 机关 block，偏橙，tool call 密集处形成连续平台。 播放到该节点时 Evidence 显示 tool、call id、inline evidence。
file_change 绿色 bridge，表示工程状态被修改。 点击后进入 artifacts / file evidence，后续可接 diff preview。
verification checkpoint flag，表示测试、构建、QA 或人工验证。 Replay 到 checkpoint 时暂停一拍，突出成功或失败状态。
error hazard，红色尖锥或障碍物。 同步 Evidence 并标记 route 中的绕路、重试、恢复行为。
assistant_message finish gate，表示本轮输出。 同步选中 Codex 气泡；可作为 replay 终点。

实施切片

  1. 01
    Atlas 数据模型

    把当前页 TaskJourney 和已加载 detail 转成稳定的 atlas node/route。

  2. 02
    Three.js 场景

    渲染 island、route、node、agent avatar，支持 resize、hover、click。

  3. 03
    Replay Controller

    Play/Pause/Reset 逐节点推进，沿路线移动 avatar。

  4. 04
    Thread/Evidence Sync

    节点选择触发 detail 预加载、thread 展开、Evidence 选中。

下一阶段

  1. A
    Context Inspector

    专门展示每轮 run 的 context packets：文件、摘要、历史消息、工具结果。

  2. B
    Boss Fight Mode

    把错误密集、重试多、token 高的任务标记为 boss fight，形成复盘入口。

  3. C
    Engineering Achievements

    只奖励真实工程行为：测试闭环、回滚恢复、最少 context 解决高复杂任务。

  4. D
    Desktop Performance

    Atlas 渲染和 ingest worker 隔离，桌面版使用本地缓存和离屏缩略图。

设计原则：游戏化只负责让复杂过程可感知、可探索、可复盘；所有节点都必须能落回真实 log、thread 和 evidence。
